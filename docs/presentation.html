Teil 3 -- Automation

GitHub Actions Workflow

# .github/workflows/publish.yml name: Daily Tennis Article on: schedule: - cron: '0 6 * * *' # 6 UTC = 7 Uhr MEZ workflow_dispatch: # manueller Start steps: - Setup Python 3.12 - pip install requests jinja2 google-genai - python3 generate.py --count 1 - git add docs/ && git commit && git push

Der Workflow laeuft auf GitHub-Servern -- dein Computer muss nicht an sein.

Teil 3 -- Automation

Fehlerbehandlung

Was passiert, wenn etwas schiefgeht?

Graceful Fallbacks
Wenn die Bild-API fehlschlaegt: Artikel wird trotzdem publiziert (ohne Bild)

Wenn die Text-API fehlschlaegt: Workflow stoppt sauber, naechster Versuch morgen
Idempotenz
Bereits generierte Artikel werden nicht ueberschrieben

Der Workflow kann jederzeit manuell neu gestartet werden ohne Datenverlust
Git-Sicherheit
git diff --staged --quiet || git commit

Commit nur wenn sich tatsaechlich etwas geaendert hat
